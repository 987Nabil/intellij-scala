When extending a class with a non-private variable, the user might suspect that a class parameter of the subclass will override the variable from the superclass, but in fact it will only shadow it. This may sometimes lead to non-trivial errors.

class Animal {
  var name: String = ""
}

class Cat(name: String) extends Animal {
  def usage = name
}

Please note that the highlight will appear only if the value is actually used.
