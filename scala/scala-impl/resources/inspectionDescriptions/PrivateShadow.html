This is about a corner case in extending a class where the user might suspect the value in the subclass overrides the one from the superclass, but in fact it is only shadowed. That sometimes leads to non-trivial errors. The Scala compiler takes the option -Xlint:private-shadow which issues a warning in such case.

The checkbox allows for controlling how to highlight this corner case. You can leave it unchecked to always highlight the suspicious shadowing, or you can check it to comply with the compiler option and let the inspection behave accordingly.


class C(var c: Int)

// `c` shadows `var c` from the `C` class
class D(c: Int) extends C(c) {
  def f: Int = c
}
